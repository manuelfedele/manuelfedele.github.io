Introduzione
#

Questo tutorial mostra come creare un tema semplice in Hugo. Si presuppone una certa familiarità con HTML, la riga di comando bash e Markdown per formattare il contenuto. Vedremo come Hugo usa i template e come organizzarli per creare un tema. Non tratteremo l’uso dei CSS per lo stile.

Inizieremo creando un nuovo sito con un template molto elementare, poi aggiungeremo alcune pagine e articoli. Con piccole variazioni su questo schema, sarà possibile creare molti tipi diversi di siti web.

In questo tutorial, i comandi da eseguire iniziano con il prompt “$”. L’output seguirà. Le righe che iniziano con “#” sono commenti esplicativi. Quando mostriamo aggiornamenti a un file, “:wq” sull’ultima riga significa salvare il file.

Alcune definizioni
#

Prima di creare un tema, è importante capire alcuni concetti.

Skin
#

Le skin sono i file responsabili dell’aspetto visivo del sito. I CSS controllano colori e font, il Javascript determina azioni e reazioni. Sono anche le regole che Hugo usa per trasformare il contenuto in HTML da servire ai visitatori.

Ci sono due modi per creare una skin. Il più semplice è crearla nella directory layouts/. In questo caso non è necessario configurare Hugo per riconoscerla, poiché il primo posto dove Hugo cerca regole e file è proprio layouts/.

La seconda opzione è crearla in una sottodirectory di themes/. In questo caso occorre sempre indicare a Hugo dove cercare la skin, il che comporta lavoro aggiuntivo.

La differenza tra le due è sottile: una skin in layouts/ non può essere personalizzata senza modificare i template e i file statici. Una skin in themes/, invece, può essere personalizzata, il che la rende più facile da condividere con altri.

Il resto di questo tutorial chiamerà “tema” una skin creata nella directory themes/.

La Home Page
#

La home page, o pagina di atterraggio, è la prima pagina che molti visitatori vedono. Corrisponde al file index.html nella directory radice del sito web. Poiché Hugo scrive i file nella directory public/, la nostra home page è public/index.html.

File di configurazione del sito
#

Quando Hugo viene eseguito, cerca un file di configurazione che contenga impostazioni che sovrascrivono i valori predefiniti per l’intero sito. Il file può usare TOML, YAML o JSON. Si preferisce usare TOML per i file di configurazione.

Hugo traduce i file Markdown in HTML. Per impostazione predefinita, si aspetta di trovare i file Markdown nella directory content/ e i template nella directory themes/. Genererà i file HTML nella directory public/.

Contenuto
#

Il contenuto è archiviato in file di testo che contengono due sezioni. La prima è il “front matter”, ovvero le meta-informazioni sul contenuto. La seconda sezione contiene il Markdown che verrà convertito in HTML.

Front Matter
#

Il front matter contiene informazioni sul contenuto. Come il file di configurazione, può essere scritto in TOML, YAML o JSON. Hugo usa i marcatori per determinare il formato: TOML è racchiuso tra “+++”, YAML tra “---” e JSON tra parentesi graffe.

Le informazioni nel front matter vengono passate al template prima che il contenuto venga renderizzato in HTML.

Markdown
#

Il contenuto è scritto in Markdown, che semplifica la creazione dei testi. Hugo lo elabora con un motore Markdown per generare l’HTML da scrivere nel file di output.

File di template
#

Hugo usa i file di template per renderizzare il contenuto in HTML. I template sono un ponte tra contenuto e presentazione. Le regole nel template definiscono quale contenuto viene pubblicato, dove e come verrà renderizzato. Il template guida la presentazione specificando lo stile da usare.

Esistono tre tipi di template: single, list e partial. Ogni tipo prende del contenuto come input e lo trasforma in base ai comandi nel template.

Hugo usa la sua conoscenza del contenuto per trovare il file di template corretto. Se non trova una corrispondenza esatta, risale di un livello e cerca da lì. Continua finché trova un template corrispondente o esaurisce le opzioni. Se non trova nessun template, usa quello predefinito del sito.

È possibile usare il front matter per influenzare la scelta del template da parte di Hugo.
