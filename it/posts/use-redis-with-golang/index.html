Usare Golang e Redis
#

Redis è un data store in-memory che può essere usato come database, cache e message broker. È noto per la sua velocità, semplicità e flessibilità. In questo articolo vedremo come usare Redis con Golang, un linguaggio di programmazione apprezzato per la sua semplicità, le sue prestazioni e il supporto alla concorrenza.

Prima di addentrarci nei dettagli, vediamo i requisiti dell’applicazione.

Requisiti
#

  • L’applicazione deve potersi connettere a un server Redis.
  • L’applicazione deve poter memorizzare e recuperare dati da Redis.
  • L’applicazione deve poter eseguire operazioni di base sulle strutture dati di Redis: stringhe, liste, set e hash.

Configurare Redis
#

Per usare Redis con Golang è necessario avere un server Redis installato e in esecuzione. È possibile scaricare l’ultima versione dal sito ufficiale e seguire le istruzioni per installarlo e avviarlo.

Una volta avviato il server Redis, si possono usare le seguenti librerie Go per connettersi ed eseguire varie operazioni:

  • redis: un client Redis di basso livello per Go.
  • radix: un client Redis di livello più alto per Go che fornisce un’API più comoda e funzionalità aggiuntive.

Connettersi a Redis
#

Per connettersi al server Redis con la libreria redis si usa la funzione Dial:

package main

import (
    "fmt"
    "log"

    "github.com/gomodule/redigo/redis"
)

func main() {
    // Connect to the Redis server
    conn, err := redis.Dial("tcp", "localhost:6379")
    if err != nil {
        log.Fatal(err)
    }
    defer conn.Close()

    // Print the Redis version
    version, err := redis.String(conn.Do("INFO", "server"))
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println(version)
}

Questo si connetterà al server Redis in esecuzione localmente sulla porta predefinita (6379) e stamperà la versione di Redis.

Per connettersi con la libreria radix si usa la funzione Dial:

package main

import (
    "fmt"
    "log"

    "github.com/mediocregopher/radix/v3"
)

func main() {
    // Connect to the Redis server
    conn, err := radix.Dial("tcp", "localhost:6379")
    if err != nil {
        log.Fatal(err)
    }
    defer conn.Close()

    // Print the Redis version
    var version string
    err = conn.Do(radix.Cmd(&version, "INFO", "server"))
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println(version)
}

Anche questo si connetterà al server Redis sulla porta predefinita (6379).

Memorizzare e recuperare dati
#

Una volta connessi al server Redis, si possono usare i vari comandi Redis per memorizzare e recuperare dati.

Stringhe
#

Per memorizzare e recuperare un valore stringa con la libreria redis:

package main

import (
    "fmt"
    "log"

    "github.com/gomodule/redigo/redis"
)

func main() {
    // Connect to the Redis server
    conn, err := redis.Dial("tcp", "localhost:6379")
    if err != nil {
        log.Fatal(err)
    }
    defer conn.Close()

    // Set the key "message" to the value "Hello, Redis!"
    _, err = conn.Do("SET", "message", "Hello, Redis!")
    if err != nil {
        log.Fatal(err)
    }

    // Retrieve the value of the key "message"
    value, err := redis.String(conn.Do("GET", "message"))
    if err != nil {
        log.Fatal(err)
    }

    // Print the value
    fmt.Println(value)
}

Questo imposterà la chiave “message” al valore “Hello, Redis!” e recupererà il valore, stampando “Hello, Redis!”.

Per memorizzare e recuperare un valore stringa con la libreria radix:

package main

import (
    "fmt"
    "log"

    "github.com/mediocregopher/radix/v3"
)

func main() {
    // Connect to the Redis server
    conn, err := radix.Dial("tcp", "localhost:6379")
    if err != nil {
        log.Fatal(err)
    }
    defer conn.Close()

    // Set the key "message" to the value "Hello, Redis!"
    err = conn.Do(radix.FlatCmd(nil, "SET", "message", "Hello, Redis!"))
    if err != nil {
        log.Fatal(err)
    }

    // Retrieve the value of the key "message"
    var value string
    err = conn.Do(radix.Cmd(&value, "GET", "message"))
    if err != nil {
        log.Fatal(err)
    }

    // Print the value
    fmt.Println(value)
}

Anche questo imposterà la chiave “message” al valore “Hello, Redis!” e lo recupererà, stampando “Hello, Redis!”.

Liste
#

Per memorizzare e recuperare una lista con la libreria redis:

package main

import (
    "fmt"
    "log"

    "github.com/gomodule/redigo/redis"
)

func main() {
    // Connect to the Redis server
    conn, err := redis.Dial("tcp", "localhost:6379")
    if err != nil {
        log.Fatal(err)
    }
    defer conn.Close()

    // Store the list ["apple", "banana", "cherry"] in the key "fruits"
    _, err = conn.Do("RPUSH", "fruits", "apple", "banana", "cherry")
    if err != nil {
        log.Fatal(err)
    }

    // Retrieve the list from the key "fruits"
    values, err := redis.Strings(conn.Do("LRANGE", "fruits", 0, -1))
    if err != nil {
        log.Fatal(err)
    }

    // Print the list
    fmt.Println(values)
}

Questo memorizzerà la lista [“apple”, “banana”, “cherry”] nella chiave “fruits” e la recupererà, stampando [“apple”, “banana”, “cherry”].

Hash
#

Per memorizzare e recuperare un hash con la libreria redis:

package main

import (
    "fmt"
    "log"

    "github.com/gomodule/redigo/redis"
)

func main() {
    // Connect to the Redis server
    conn, err := redis.Dial("tcp", "localhost:6379")
    if err != nil {
        log.Fatal(err)
    }
    defer conn.Close()

    // Store the hash {"name": "Alice", "age": "30"} in the key "person"
    _, err = conn.Do("HMSET", "person", "name", "Alice", "age", "30")
    if err != nil {
        log.Fatal(err)
    }

    // Retrieve the hash from the key "person"
    values, err := redis.Strings(conn.Do("HGETALL", "person"))
    if err != nil {
        log.Fatal(err)
    }

    // Print the hash
    fmt.Println(values)
}

Questo memorizzerà l’hash {“name”: “Alice”, “age”: “30”} nella chiave “person” e lo recupererà, stampando {“name”, “Alice”, “age”, “30”}.

Conclusione
#

In questo articolo abbiamo visto come usare le librerie redis e radix per connettersi a un server Redis e memorizzare e recuperare vari tipi di dati in Go. Redis è un data store in-memory potente e veloce, adatto a un’ampia gamma di applicazioni. Le librerie redis e radix rendono semplice usarlo in Go.
