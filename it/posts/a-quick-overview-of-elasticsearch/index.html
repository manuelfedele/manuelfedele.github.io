Introduzione
#

Elastic, noto anche come Elasticsearch, è un potente motore di ricerca e analisi che consente di indicizzare, cercare e analizzare grandi volumi di dati rapidamente e in quasi tempo reale. È open-source e costruito sulla libreria Lucene. In questo articolo vedremo le basi di Elastic e come iniziare a utilizzarlo.

  1. Installazione
    #

    Il primo passo per usare Elastic è installarlo sul proprio sistema. È possibile scaricare l’ultima versione dal sito ufficiale (https://www.elastic.co/downloads/elasticsearch) e seguire le istruzioni fornite. Una volta installato, si può avviare il servizio Elastic sulla propria macchina e accedervi tramite un’interfaccia web all’indirizzo http://localhost:9200.

  2. Indicizzazione dei dati
    #

    In Elastic, i dati sono archiviati in indici, simili alle tabelle di un database relazionale. Per indicizzare i dati occorre creare un indice e poi aggiungere documenti. Si può creare un indice tramite le API di Elastic, ad esempio inviando una richiesta PUT al seguente endpoint: http://localhost:9200/nome_indice. Una volta creato l’indice, si possono aggiungere documenti con una richiesta POST allo stesso endpoint, allegando i propri dati JSON.

  3. Ricerca dei dati
    #

    Una volta indicizzati i dati, è possibile effettuare ricerche tramite le API di Elastic. La ricerca più elementare è la match query, che restituisce tutti i documenti corrispondenti a un termine specificato. Si può eseguire una match query con una richiesta GET al seguente endpoint: http://localhost:9200/nome_indice/_search?q=termine_cercato. Sono disponibili anche opzioni di ricerca avanzate come filtri, aggregazioni e highlighting.

  4. Analisi dei dati
    #

    Elastic offre anche diversi strumenti per analizzare i dati, tra cui la possibilità di creare visualizzazioni e dashboard con Kibana. Kibana è un prodotto separato ma si integra facilmente con Elastic. Con Kibana è possibile creare grafici, tabelle e mappe per visualizzare i propri dati e ricavarne informazioni utili.

Indicizzazione dei dati
#

  1. Creare un indice
    #

    Il modo più semplice per creare un indice in Elasticsearch è inviare una richiesta PUT all’endpoint API corrispondente. Ad esempio, per creare un indice chiamato “customers”:
    curl -XPUT http://localhost:9200/customers
    Questo creerà un indice con le impostazioni e il mapping predefiniti.

  2. Personalizzare i mapping
    #

    Quando si crea un indice, Elasticsearch genera automaticamente un mapping che definisce come trattare i dati. È possibile personalizzarlo inviando una richiesta PUT con un documento JSON che descrive i mapping desiderati. Ad esempio, per creare un indice “customers” con mapping per i campi “name”, “age” e “address”:
    curl -XPUT http://localhost:9200/customers -H 'Content-Type: application/json' -d '
{
     "mappings": {
         "properties": {
             "name": { "type": "text" },
             "age": { "type": "integer" },
             "address": { "type": "text" }
         }
     }
 }'

  3. Aggiornare i mapping
    #

    Se l’indice esiste già e si vuole aggiornarne i mapping, si può usare l’endpoint _mapping con una richiesta PUT. Ad esempio, per aggiornare i mapping dell’indice “customers”:
    curl -XPUT http://localhost:9200/customers/_mapping -H 'Content-Type: application/json' -d '
{
     "properties": {
         "name": { "type": "text" },
         "age": { "type": "integer" },
         "address": { "type": "text" }
     }
 }'

  4. Eliminare un indice
    #

    Se un indice non è più necessario, si può eliminare con una richiesta DELETE. Ad esempio:
    curl -XDELETE http://localhost:9200/customers

Ricerca dei dati
#

  1. Ricerca semplice
    #

    La ricerca più elementare in Elasticsearch è la match query, che restituisce tutti i documenti corrispondenti a un termine. Ad esempio, per cercare tutti i documenti nell’indice “customers” con il termine “John” nel campo “name”:
    curl -XGET http://localhost:9200/customers/_search -H 'Content-Type: application/json' -d '
{
     "query": {
         "match": {
             "name": "John"
         }
     }
 }'

  2. Ricerca avanzata
    #

    Elasticsearch offre anche molte opzioni avanzate come filtri, aggregazioni e highlighting. Ad esempio, per cercare documenti nell’indice “customers” con il termine “John” nel campo “name” e con età superiore a 30:
    curl -XGET http://localhost:9200/customers/_search -H 'Content-Type: application/json' -d '
{
     "query": {
         "bool": {
             "must": {
                 "match": {
                     "name": "John"
                 }
             },
             "filter": {
                 "range": {
                     "age": {
                         "gt": 30
                     }
                 }
             }
         }
     }
 }'

  3. Ricerca con aggregazioni
    #

    È possibile eseguire aggregazioni sui risultati di ricerca per ottenere informazioni sui dati. Ad esempio, per cercare tutti i documenti nell’indice “customers” e raggrupparli per il campo “age”:
    curl -XGET http://localhost:9200/customers/_search -H 'Content-Type: application/json' -d '
     {
         "query": {
             "match_all": {}
         },
         "aggs": {
             "group_by_age": {
                 "terms": {
                     "field": "age"
                 }
             }
         }
     }'

  4. Ricerca con highlighting
    #

    È possibile usare l’highlighting per evidenziare i termini cercati nei risultati. Ad esempio, per cercare documenti con il termine “John” e evidenziarlo:
    curl -XGET http://localhost:9200/customers/_search -H 'Content-Type: application/json' -d '
{
     "query": {
         "match": {
             "name": "John"
         }
     },
     "highlight": {
         "fields": {
             "name": {}
         }
     }
 }'

Indici aggregati
#

A volte è necessario eseguire aggregazioni senza mai usare i “dati grezzi”. In questi casi, creare un indice aggregato può migliorare le prestazioni. Un’aggregazione è un modo per raggruppare, filtrare o calcolare statistiche sui dati. In Elasticsearch esistono diversi tipi di aggregazioni: terms, range, date histogram e molti altri. Ad esempio, si può usare un’aggregazione terms per raggruppare i dati per un campo specifico come l’età, o un’aggregazione range per filtrare i dati in base a un intervallo di valori.

  1. Creare indici aggregati
    #

    Si può creare un indice e definire aggregazioni inviando una richiesta PUT all’endpoint corrispondente, includendo le aggregazioni nel corpo della richiesta. Ad esempio, per creare un indice “sales” con un’aggregazione sul campo “price”:

    curl -XPUT http://localhost:9200/sales -H 'Content-Type: application/json' -d '
{
     "mappings": {
         "properties": {
             "price": { "type": "double" },
             "date": { "type": "date" }
         }
     },
     "aggs": {
         "avg_price": {
             "avg": {
                 "field": "price"
             }
         }
     }
 }'

  2. Usare le aggregazioni
    #

    Una volta creato l’indice aggregato, si possono eseguire query di ricerca per recuperare i risultati aggregati. Ad esempio, per recuperare il prezzo medio di tutti i documenti nell’indice “sales”:

    curl -XGET http://localhost:9200/sales/_search -H 'Content-Type: application/json' -d '
{
     "aggs": {
         "avg_price": {
             "avg": {
                 "field": "price"
             }
         }
     }
 }'

  3. Combinare aggregazioni
    #

    È anche possibile combinare più aggregazioni per ottenere informazioni più dettagliate. Ad esempio, si può usare un’aggregazione terms per raggruppare i dati per tipo di prodotto e poi un’aggregazione range per filtrare per intervallo di prezzo.

     curl -XGET http://localhost:9200/sales/_search -H 'Content-Type: application/json' -d '
 {
     "aggs": {
         "product_type": {
             "terms": {
                 "field": "product_type"
             },
             "aggs": {
                 "price_range": {
                     "range": {
                         "field": "price",
                         "ranges": [
                             { "to": 10 },
                             { "from": 10, "to": 20 },
                             { "from": 20 }
                         ]
                     }
                 }
             }
         }
     }
 }'

    In questo esempio, la query raggruppa i documenti per “product_type” e poi li filtra per fascia di prezzo, permettendo di capire come si comportano i diversi tipi di prodotto in base al prezzo.

    Un altro esempio potrebbe essere:

    curl -XGET http://localhost:9200/sales/_search -H 'Content-Type: application/json' -d '
{
    "aggs": {
        "by_customer_id": {
            "terms": {
                "field": "customer_id"
            },
            "aggs": {
                "avg_price": {
                    "avg": {
                        "field": "price"
                    }
                }
            }
        }
    }
}'

    In questo modo è possibile capire come si comportano i diversi clienti in base al prezzo medio dei loro acquisti.

In conclusione, Elastic è uno strumento potente per indicizzare, cercare e analizzare grandi volumi di dati. È facile da avviare e offre un’ampia gamma di funzionalità. Quello descritto qui è solo la punta dell’iceberg. Vale la pena esplorarlo e scoprire come può aiutare a soddisfare le proprie esigenze di analisi su grandi dati.
